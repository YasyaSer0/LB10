### Glossary of Basic English Terms

**File ownership - the concept that each file has a user owner and a group owner in the system.**

**User owner - the user who owns a file and has primary control over it.**

**Group owner - the group associated with a file that defines access for multiple users.**

**Permissions - rules that define how files and directories can be accessed.**

**Read permission (r) - allows viewing the contents of a file or listing a directory.**

**Write permission (w) - allows modifying a file or managing files in a directory.**

**Execute permission (x) - allows running a file as a program or accessing a directory.**

**ls - a command used to list files and directories.**

**ls -l - displays detailed information about files including permissions and ownership.**

**ls -la - shows all files including hidden ones with detailed information.**

**Hidden files - files that start with a dot (.) and are not displayed by default.**

**UID - a unique identifier assigned to each user.**

**GID - a unique identifier assigned to each group.**

**id - displays user identity information including UID, GID, and groups.**

**groups - shows all groups that a user belongs to.**

**newgrp - changes the current primary group of a user session.**

**Primary group - the default group assigned to a user.**

**Supplementary group - additional groups a user is a member of.**

**chmod - a command used to change file permissions.**

**Symbolic mode - a method of changing permissions using symbols (u, g, o, +, -, =).**

**Numeric (octal) mode - a method of setting permissions using numbers (e.g., 755, 644).**

**chown - a command used to change the owner of a file.**

**chgrp - a command used to change the group owner of a file.**

**Default permissions - permissions assigned automatically when a file is created.**

**umask - a command that defines default permission values for new files.**

**setuid - a special permission that allows execution of a file with the owner's privileges.**

**setgid - a special permission that allows files to inherit group ownership.**

**Sticky bit - a special permission that allows only file owners to delete files in a directory.**

**Executable file - a file that can be run as a program.**

**Directory - a container used to store files and other directories.**

**Shell - a command-line interface used to interact with the operating system.**

### 1. Яке призначення команди id?

Команда id використовується для перегляду інформації про користувача. Вона відображає UID (ідентифікатор користувача), GID (ідентифікатор групи), а також усі групи, до яких належить користувач.

### 3. Як змінити власника групи?

Змінити групу-власника файлу можна за допомогою команди chgrp:
```bash
chgrp назва_групи ім'я_файлу
```
Або за допомогою команди chown:
```bash
chown :назва_групи ім'я_файлу
```

### 5. Для чого використовуються дозволи Setuid та Setgid?

Це спеціальні дозволи доступу:
- Setuid дозволяє запускати файл з правами його власника (часто root).
- Setgid дозволяє запускати файл з правами групи або автоматично призначає групу директорії новим файлам.

Використовуються для контролю доступу до системних ресурсів.

### 6. Для чого потрібен Sticky Bit? Наведіть приклади.

Sticky Bit - це спеціальний дозвіл для директорій, який дозволяє видаляти файли лише їх власнику.

Застосовується у спільних директоріях, наприклад /tmp, де багато користувачів створюють файли.

| `touch priv-dir/priv-file`          | Створює порожній файл priv-file у директорії priv-dir.                       |
| `touch pub-dir/pub-file`            | Створює порожній файл pub-file у директорії pub-dir.                         |
| `ls -l priv-dir`                    | Відображає детальну інформацію про файли в директорії priv-dir.              |
| `ls -l pub-dir`                     | Відображає детальну інформацію про файли в директорії pub-dir.               |
| `ls -la`                            | Показує всі файли (включаючи приховані) з детальною інформацією.             |
| `ls -ld priv-dir`                   | Відображає інформацію про саму директорію priv-dir.                          |
| `chmod o-rx priv-dir`               | Забирає права читання і виконання у інших користувачів.                      |
| `chmod a+x file`                    | Додає право виконання для всіх користувачів.                                 |
| `chmod g-w file`                    | Забирає право запису у групи.                                                |
| `chmod go+r file`                   | Додає право читання для групи та інших користувачів.                         |
| `chmod o=rwx`                       | Встановлює повні права доступу для інших користувачів.                       |
| `chmod o+w pub-dir`                 | Додає право запису для інших користувачів у директорії.                      |
| `chmod g-rw,o-r priv-dir/priv-file` | Забирає права у групи та інших користувачів для файлу.                       |
| `chmod a=rw pub-dir/pub-file`       | Встановлює однакові права читання і запису для всіх.                         |
| `echo "date" > test.sh`             | Створює файл test.sh і записує в нього команду.                              |
| `./test.sh`                         | Виконує скрипт test.sh.                                                      |
| `ls -l test.sh`                     | Показує права доступу до файлу test.sh.                                      |
| `chmod u+x test.sh`                 | Додає право виконання для власника файлу.                                    |
| `stat test.sh`                      | Виводить детальну інформацію про файл (включаючи права в числовому вигляді). |
| `chmod 775 test.sh`                 | Встановлює права доступу у числовому форматі (rwxrwxr-x).                    |
| `su -`                              | Перемикає користувача на root.                                               |
| `ls -ld pub-dir`                    | Відображає інформацію про директорію pub-dir.                                |
| `ls -l pub-dir/pub-file`            | Показує інформацію про файл pub-file.                                        |
| `chown root:root pub-dir`           | Змінює власника і групу директорії на root.                                  |
| `chown bin pub-dir/pub-file`        | Змінює власника файлу на користувача bin.                                    |
| `ls -ld priv-dir`                   | Показує інформацію про директорію priv-dir.                                  |
| `ls -l priv-dir/priv-file`          | Показує інформацію про файл priv-file.                                       |
| `chgrp -R users priv-dir`           | Рекурсивно змінює групу для директорії і всіх її файлів.                     |

### Контрольні запитання

#### 1. Наведіть приклади зміни прав доступу символічним методом (Symbolic Method)
```bash
chmod u+rwx file.txt     # додати всі права власнику
chmod g+rx file.txt      # дати групі читання і виконання
chmod o-rwx file.txt     # забрати всі права у інших
chmod u-x file.txt       # забрати право виконання у власника
chmod g=w file.txt       # залишити групі тільки запис
```

Скрін - зміна прав символічним методом

#### 3. Яке призначення команди umask?

Команда umask визначає права доступу за замовчуванням для нових файлів і директорій.
```bash
umask
touch file1
ls -l file1

umask 077
touch file2
ls -l file2
```

#### 5. Чи можна виконати файл без права читання (--x)?
```bash
echo "date" > exec.sh
chmod 111 exec.sh
./exec.sh
```

Результат:
```bash
Permission denied
```
Пояснення:

Ні, файл виконати не можна, тому що оболонці потрібне право читання, щоб прочитати його вміст.

#### 6. Чи зберігаються права доступу після завершення сесії?

Так, права доступу зберігаються, тому що вони записані у файловій системі.

Вони не залежать від сесії користувача.

#### 7. Чи є шаблон прав доступу? Як змінити?

Так, використовується umask.
```bash
umask 002
touch file1

umask 077
touch file2
```

umask 002 → файл: rw-rw-r--
umask 077 → файл: rw-------

#### 11. Що буде при видаленні файлів і посилань?

Приклад:
```bash
echo "data" > fileA
ln fileA hardA
ln -s fileA softA
rm fileA
```

Результат:
- hardA → працює (дані збережені)
- softA → битий (broken link)

Якщо видалити:

1. Оригінальний файл
- hard link → працює
- soft link → не працює

2. Символічне посилання
- нічого не змінюється

3. Жорстке посилання
- файл залишається, якщо є інші посилання
